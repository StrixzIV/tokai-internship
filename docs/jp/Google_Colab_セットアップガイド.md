# Google Colab ノートブック セットアップ & 使用ガイド

このドキュメントでは、EyeLink `.asc` ファイルの解析および PCHIP 瞬き補間を実行するための Google Colab ノートブックのセットアップ手順と使用方法について説明します。

コラボノートブックはリポジトリ内の以下の場所に保存されています：
`src/python/led-flash/colab_parsing_and_interpolation.ipynb`

---

## 1. Google Colab でノートブックを開く

1. ウェブブラウザを開き、[Google Colab](https://colab.research.google.com) にアクセスします。
2. Google アカウントでサインインします。
3. ポップアップダイアログで **アップロード (Upload)** タブを選択します。
4. ローカルリポジトリから `colab_parsing_and_interpolation.ipynb` ファイルをドラッグ＆ドロップするか、**ファイルの選択 (Choose File)** をクリックしてファイルを選択します。

---

## 2. ステップ・バイ・ステップの使用方法

ノートブックは4つの主なステップに分かれています。各コードセルの左側にある **実行 (▶)** ボタンをクリックして、セルを順番に実行してください。

### ステップ 1: EyeLink `.asc` ファイルのアップロード
- **Step 1** のコードセルを実行します。
- 表示される **ファイルを選択 (Choose Files)** ボタンをクリックします。
- EyeLink `.edf` ファイルからエクスポートした生の `.asc` ファイル（例: `260611.asc`）を選択します。
- ファイルが Colab 仮想環境のメモリにアップロードされます。

### ステップ 2: パイプラインの定義と実行
- **Step 2** のコードセルを実行して、`parse_and_clean_asc` 関数を定義し、アップロードされたファイルを処理します。
- **スライディングウィンドウの調整 (`SLIDING_WINDOW_SIZE`)**:
  - デフォルト値は `200` です（1000 Hz のサンプリングレートで 200 ms のウィンドウに相当）。
  - *値を大きくする場合*: プロットで瞬きアーティファクトの両端（スロープ部分）がゼロに向かって急降下している場合は、値を `300` 以上に大きくして、瞼の開閉に伴う歪みデータを完全にカバーしてください。
  - *値を小さくする場合*: 正常なベースライン瞳孔径データが不必要にカットされてしまっている場合は、値を `100` または `150` に小さくしてください。
  - 変更する場合は、セル内のコードで `SLIDING_WINDOW_SIZE = 200` の値を直接編集してからセルを実行してください。

### ステップ 3: 瞳孔反応の視覚化
- **Step 3** のコードセルを実行し、瞳孔径の経時変化をプロットします。
- **検証ポイント**:
  - **薄グレーの線**: 生の瞳孔径データ（瞬き中は `0` に急降下している部分）。
  - **濃い緑の線**: クリーニング済みの瞳孔径データ（PCHIP補間により瞬きギャップが滑らかに繋がっている部分）。
  - **薄青の背景ブロック**: 青色LED刺激がアクティブ（`blue_active = 1`）だった時間帯を示します。
  - 緑の線が、オーバーシュート（波打ち）を起こさずに瞬きの隙間を滑らかに補間できているか確認してください。

### ステップ 4: エクスポートとダウンロード
- **Step 4** のコードセルを実行します。
- 解析およびクリーニングされたデータが、CSVとExcelの双方の形式でエクスポートされます。
  - **解析生データ**: `<filename>_parsed.csv` & `<filename>_parsed.xlsx`
  - **解析＆補間済みデータ**: `<filename>_cleaned.csv` & `<filename>_cleaned.xlsx`
- Colab の機能により、これら4つのファイルがパソコンのローカルマシンに自動的に順次ダウンロードされます。

## 3. 出力ファイルのデータ構造

エクスポートされたファイルには、以下の列（カラム）が含まれています。

| 列名 (Column) | データ型 | 説明 |
| :--- | :--- | :--- |
| `timestamp` | 整数型 (Int) | EyeLink 記録時の生のタイムスタンプ (ms)。 |
| `elasped_ms` | 整数型 (Int) | 記録開始時点からの経過ミリ秒。 |
| `elasped_sec` | 浮動小数点型 (Float) | 記録開始時点からの経過秒。 |
| `blue_active` | バイナリ型 (0/1) | 青色LED刺激の状態（1でオン、0でオフ）。Eyelink の `BUTTON` イベントから再構築。 |
| `gaze_x` | 浮動小数点型 (Float) | 視線の水平座標（瞬き中は `NaN`）。 |
| `gaze_y` | 浮動小数点型 (Float) | 視線の垂直座標（瞬き中は `NaN`）。 |
| `pupil_size` | 浮動小数点型 (Float) | 生の瞳孔径（単眼・左目、瞬き中は `0.0`）。 |
| `cleaned_pupil_size` | 浮動小数点型 (Float) | PCHIP瞬き補間後の瞳孔径（`_cleaned` ファイルにのみ含まれます）。 |
